Vagrant · Ansible · VMware Desktop · Debian 12

🎯

Objectif: créer deux VMs Debian 12 avec une pile LAMP complète (Apache + MariaDB + PHP), provisionnées automatiquement.

  • VM headless: serveur local en SSH
  • VM GNOME: bureau graphique

Ce tutoriel t'explique ce que fait chaque bloc et pourquoi. À la fin de chaque section, un fichier complet est disponible pour vérifier ton travail.

1) Prérequis

Vagrant

Vagrant est l’outil qui va piloter la création de ta VM. Il lit un Vagrantfile et gère tout le cycle de vie: création, démarrage, provisioning, suppression.

Installe-le depuis: https://developer.hashicorp.com/vagrant/install

Vérifie ensuite:

vagrant --version

VMware Desktop

C’est l’hyperviseur qui va exécuter la VM sur ta machine.

Installe VMware Workstation (Windows / Linux) ou VMware Fusion (macOS), selon ton système.

Plugin Vagrant VMware

Vagrant ne parle pas à VMware nativement: il faut un plugin.

vagrant plugin install vagrant-vmware-desktop

Puis vérifie:

vagrant plugin list
# vagrant-vmware-desktop doit apparaître
⚠️

Si vagrant up échoue avec provider 'vmware_desktop' not found, c’est que VMware Desktop n’est pas installé ou que le plugin est manquant.

2) Structure du projet

Crée les dossiers et fichiers vides suivants:

ton-projet/
├── VM headless/
│   ├── Vagrantfile
│   └── deploy.yml
└── VM GNOME/
    ├── Vagrantfile
    └── deploy.yml

Chaque dossier est indépendant: tu peux utiliser l’un sans l’autre.

La séparation est volontaire:

  • le Vagrantfile définit la machine (ressources, réseau, provider)
  • le deploy.yml définit ce qui tourne dedans (Apache, PHP, MariaDB)

3) Le Vagrantfile headless — bloc par bloc

Choisir la box de base

Vagrant.configure("2") do |config|
  config.vm.box = "generic/debian12"

Vagrant.configure("2") ouvre le bloc de configuration (le "2" est la version de l’API Vagrant, toujours "2" aujourd’hui).

config.vm.box désigne l’image de base de la VM — ici Debian 12. Vagrant télécharge cette box automatiquement depuis Vagrant Cloud si elle n’est pas encore en cache.

Nommer la machine

config.vm.hostname = "lamp-server"

Ce nom identifie la machine sur le réseau interne. Il apparaît dans le prompt SSH une fois connecté (demo@lamp-server).

Adapte le à ton projet.

Configurer le provider VMware

config.vm.provider "vmware_desktop" do |v|
  v.gui    = false
  v.vmx["displayName"] = "LAMP-Server"
  v.cpus   = 2
  v.memory = 2048
end
  • v.gui = false: la VM tourne en arrière-plan, sans fenêtre. Accès SSH uniquement. Pour la version GNOME, on passera ce paramètre à true.
  • v.vmx["displayName"]: le nom affiché dans l’interface VMware.
  • v.cpus / v.memory: les ressources allouées. 2 CPU et 2 Go sont suffisants pour du LAMP headless.

Configurer le réseau

config.vm.network "private_network", type: "dhcp"

Cela crée un réseau privé entre ta machine hôte et la VM. La VM reçoit une IP automatique en DHCP.

C’est ce réseau qui te permettra d’accéder à http://<ip-de-la-vm>/info.php depuis ton navigateur.

Partager le dossier courant

config.vm.synced_folder ".", "/vagrant"

Cela monte le dossier courant (où se trouve le Vagrantfile) dans /vagrant à l’intérieur de la VM. C’est ce qui permet à Ansible d’accéder au fichier deploy.yml.

Provisioning shell: préparer la machine

config.vm.provision "shell", inline: <<-SHELL
  sed -i 's/XKBLAYOUT=.*/XKBLAYOUT="fr"/' /etc/default/keyboard
  dpkg-reconfigure -f noninteractive keyboard-configuration 2>/dev/null || true

  if ! id "demo" &>/dev/null; then
    useradd -m -s /bin/bash demo
    echo "demo:demo" | chpasswd
    usermod -aG sudo demo
  fi
SHELL

Ce bloc shell s’exécute avant Ansible. Il fait deux choses:

Clavier en AZERTY: Debian 12 démarre par défaut avec un clavier QWERTY. La première ligne modifie le fichier de configuration clavier pour passer en français.

Création d’un utilisateur: on crée l’utilisateur demo avec le mot de passe demo, un dossier home (-m) et le shell bash. La condition if ! id "demo" évite de recréer l’utilisateur si le provisioning est relhancé.

Change demo par ton nom d’utilisateur et demo:demo par utilisateur:motdepasse.

Provisioning Ansible: installer la stack

config.vm.provision "ansible_local" do |ansible|
  ansible.playbook           = "deploy.yml"
  ansible.install            = true
  ansible.compatibility_mode = "2.0"
end

Après le shell, Vagrant déclenche Ansible.

  • ansible_local: Ansible est installé et exécuté dans la VM elle-même. Tu n’as rien à installer côté hôte.
  • ansible.install = true: Vagrant gère l’installation d’Ansible dans la VM automatiquement.
  • ansible.playbook: le fichier YAML à exécuter (il est accessible via /vagrant/deploy.yml grâce au dossier partagé).

4) Le playbook Ansible — bloc par bloc

Le fichier deploy.yml est un playbook Ansible: une liste de tâches à exécuter sur la machine cible.

En-tête du playbook

---
- name: "Installation de la pile LAMP"
  hosts: all
  become: yes
  become_method: sudo
  gather_facts: yes
  • ---: début d’un fichier YAML.
  • hosts: all: le playbook s’applique à toutes les machines de l’inventaire (ici, la VM Vagrant).
  • become: yes: les tâches s’exécutent en tant que root via sudo. Indispensable pour installer des paquets.
  • gather_facts: yes: Ansible collecte des informations sur la machine (OS, architecture…) au début. Utile pour écrire des tâches conditionnelles.

Tâche 1 — Mettre à jour APT

- name: "1. Mise à jour du cache APT"
  ansible.builtin.apt:
    update_cache: yes
    cache_valid_time: 3600

Avant d’installer quoi que ce soit, il faut rafraîchir la liste des paquets disponibles.

cache_valid_time: 3600 évite de mettre à jour le cache si c’est déjà été fait dans l’heure. Pratique quand on relance vagrant provision plusieurs fois.

Tâche 2 — Installer les paquets LAMP

- name: "2. Installation des paquets LAMP"
  ansible.builtin.apt:
    name:
      - apache2
      - mariadb-server
      - mariadb-client
      - php
      - libapache2-mod-php
      - php-mysql
      - php-cli
      - php-mbstring
      - php-xml
      - php-curl
    state: present

Chaque ligne installe un paquet:

  • apache2: le serveur web
  • mariadb-server + mariadb-client: la base de données
  • php + libapache2-mod-php: PHP et son intégration avec Apache
  • les php-*: extensions courantes (connexion MySQL, manipulation de chaînes, XML, requêtes HTTP)

state: present signifie “installe si pas déjà présent”. C’est ce qui rend Ansible idempotent: tu peux relancer la tâche autant de fois que tu veux, elle ne réinstallère pas si c’est déjà en place.

Tâche 3 — Démarrer les services

- name: "3. Démarrage des services"
  ansible.builtin.service:
    name: " item "
    state: started
    enabled: yes
  loop:
    - apache2
    - mariadb

Installer un paquet ne démarre pas automatiquement le service. Cette tâche le fait explicitement.

  • state: started: démarre le service maintenant
  • enabled: yes: le service redémarre automatiquement après un reboot
  • loop: la même tâche est appliquée pour apache2 et mariadb. item prend la valeur courante de la boucle.

Tâche 4 — Page de vérification PHP

- name: "4. Création d'une page PHP de test"
  ansible.builtin.copy:
    dest: /var/www/html/info.php
    content: |
      <?php
      $apache  = function_exists('apache_get_version') ? apache_get_version() : 'Apache actif';
      $php     = phpversion();
      $db      = new mysqli('localhost', 'root', '');
      $mariadb = $db->server_info ?? 'Non connecté';
      ?>
      <!DOCTYPE html>
      <html>
      <head><meta charset="utf-8"><title>LAMP OK</title></head>
      <body style="font-family:sans-serif;max-width:500px;margin:40px auto">
        <h1>✅ Pile LAMP opérationnelle</h1>
        <ul>
          <li><strong>Apache:</strong> <?= $apache ?></li>
          <li><strong>PHP:</strong> <?= $php ?></li>
          <li><strong>MariaDB:</strong> <?= $mariadb ?></li>
        </ul>
      </body>
      </html>

Cette tâche crée un fichier PHP dans le dossier web d’Apache.

ansible.builtin.copy avec content: écrit directement le contenu du fichier sans avoir besoin d’un fichier source. Le | en YAML signifie “contenu littéral multiligne”.

La page PHP interroge:

  • la version d’Apache via apache_get_version()
  • la version de PHP via phpversion()
  • la version de MariaDB via une connexion mysqli

Accès: http://<ip-de-la-vm>/info.php

5) Le Vagrantfile GNOME — les différences

Le Vagrantfile de la VM GNOME est très proche de la version headless. Voici uniquement ce qui change.

Interface graphique activée

v.gui = true

Passé à true, VMware ouvre la fenêtre graphique automatiquement au vagrant up.

Plus de mémoire

v.memory = 4096

GNOME consomme plus de RAM qu’un serveur headless. 4 Go est le minimum raisonnable pour un bureau fluide.

Installation de GNOME dans le shell

# Installation de GNOME
apt-get update -qq
DEBIAN_FRONTEND=noninteractive apt-get install -y gnome-core gdm3

Le paquet gnome-core installe l’environnement de bureau GNOME dans sa version minimale. gdm3 est le gestionnaire de session (login graphique).

DEBIAN_FRONTEND=noninteractive évite les boites de dialogue interactives pendant l’installation: sans ça, certains paquets demandent des confirmations qui bloquent le provisioning.

Autologin configuré

    mkdir -p /etc/gdm3
    cat > /etc/gdm3/daemon.conf <<EOF
[daemon]
AutomaticLoginEnable=true
AutomaticLogin=demo
EOF

Ce bloc écrit le fichier de configuration de GDM (le gestionnaire de login GNOME). Il configure la connexion automatique au démarrage sans mot de passe pour l’utilisateur demo.

Si tu changes le nom d’utilisateur, mets-le aussi ici.

    sed -i 's/XKBLAYOUT=.*/XKBLAYOUT="fr"/' /etc/default/keyboard
    dpkg-reconfigure -f noninteractive keyboard-configuration 2>/dev/null || true

    if ! id "demo" &>/dev/null; then
      useradd -m -s /bin/bash demo
      echo "demo:demo" | chpasswd
      usermod -aG sudo demo
    fi

    apt-get update -qq
    DEBIAN_FRONTEND=noninteractive apt-get install -y gnome-core gdm3

    mkdir -p /etc/gdm3
    cat > /etc/gdm3/daemon.conf <<EOF
[daemon]
AutomaticLoginEnable=true
AutomaticLogin=demo
EOF
  SHELL

  config.vm.provision "ansible_local" do |ansible|
    ansible.playbook           = "deploy.yml"
    ansible.install            = true
    ansible.compatibility_mode = "2.0"
  end

end
